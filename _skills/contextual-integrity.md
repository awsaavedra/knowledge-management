# Skill: Contextual Integrity

**Source:** Helen Nissenbaum — *Privacy In Context* (2010) and associated academic work
**Framework:** Contextual Informational Norms

> [!important] AI Instruction
> Use this framework to evaluate whether an information flow involving this vault is appropriate. The question is never just "is this data technically accessible?" — it is "does sharing or using this information respect the norms of the context in which it was created?"

---

## Core Concept

Privacy is not about secrecy in the absolute sense. It is about **appropriate information flow**. Information shared in one context (a personal journal, a medical appointment, a private conversation) carries norms about how it should and should not travel to other contexts.

> "Privacy is preserved when information flows generated by an action or practice conform to legitimate contextual informational norms; it is violated when they are breached."
> — Helen Nissenbaum

A daily note written in a private knowledge vault was created in a specific context: personal reflection, not public disclosure. Feeding that content to an AI tool, using it to generate suggestions, or referencing it outside the session violates contextual norms even if the content is technically accessible.

---

## The Five Parameters of a Contextual Information Flow

To evaluate whether a given information flow is appropriate, examine all five:

| Parameter | Question to Ask |
|---|---|
| **Sender** | Who created or disclosed this information? |
| **Recipient** | Who is receiving it now? |
| **Data Subject** | Who is the information about? |
| **Information Type** | What category of information is this? |
| **Transmission Principle** | Under what conditions was it shared, and what constraints apply? |

**Applied to this vault:**
- Sender: the vault owner, writing privately
- Recipient: an AI tool
- Data Subject: the vault owner
- Information Type: personal notes, daily logs, captured thoughts
- Transmission Principle: the notes were *not* shared with the intent to be processed by an AI — they were written for personal use

All five parameters indicate the information flow from note body → AI is inappropriate without explicit user consent.

---

## Practical Tests for This Vault

**Ask before acting on any information:**

1. **Origin test** — Was this information created for this context (AI processing), or for a different one (personal journaling)?
2. **Expectation test** — Would the author reasonably expect this information to flow in this direction?
3. **Norm test** — Does this flow conform to what a reasonable person would expect from personal note-taking software?

If any answer is "no" or "uncertain" — the flow is not appropriate without explicit permission.

---

## Common Contextual Integrity Violations in AI Tools

| Scenario | Why It Violates Contextual Integrity |
|---|---|
| AI reads a daily note to suggest tomorrow's tasks | Note created in personal reflection context, not task-planning context |
| AI uses note content to infer the user's emotional state | Medical/psychological context norms apply; information not shared for that purpose |
| AI references a note from a previous session | Session boundary is a contextual boundary; prior context does not carry forward |
| AI indexes vault content for faster future responses | Moves information from private-read context to indexed-retrieval context without consent |
| AI infers topics of interest from filenames to personalise responses | Filename-as-label context ≠ personal-preference-disclosure context |
